# Бизнес-правила

Сводка инвариантов проекта, извлеченная из [technical-specification.md](technical-specification.md). При конфликте приоритет у ТЗ. При изменении любого правила обновить оба файла и [changelog.md](changelog.md).

## Жизненный цикл объявления

- Статусы: **Черновик → На модерации → Опубликовано / Отклонено → В архиве**.
- Черновик формирует AI; поставщик отправляет его на модерацию или правит через Web (CTA URL).
- Через Web редактируются только черновики и отклонённые объявления; сохранение сразу отправляет на модерацию, и для отправки все бизнес-поля (тип, категория, описание, локация, цена) обязательны. Объявление «На модерации» и архивное — только просмотр.
- Публикация возможна только после одобрения оператором в админке.
- Причину отклонения поставщик видит только в Web-интерфейсе, не в WhatsApp.
- В поисковом матчинге участвуют только объявления в статусе «Опубликовано» с неистекшим `expires_at`.

## 30-дневный цикл актуальности

- `expires_at` = 30 дней с момента публикации или последнего подтверждения.
- За сутки до истечения поставщику отправляется Utility Template Message с кнопками [Да, актуально] / [Нет, в архив].
- [Да] — `expires_at` продлевается на 30 дней, статус остается «Опубликовано».
- [Нет] или отсутствие ответа в течение 24 часов — объявление автоматически уходит «В архив» и исчезает из выдачи.

## AI-ассистент

- AI включается только на шагах сценария с типом «Запрос ввода» (конструктор передает управление).
- Обязательные поля объявления: тип, категория, описание, локация, цена. При нехватке данных AI дозапрашивает их.
- **Лимит уточнений: 2–3 попытки.** При исчерпании AI создает Черновик с пустыми полями и принудительно отправляет CTA URL для ручного заполнения.
- Сложное редактирование в чате не реализуется — только через Web по CTA URL.
- Матчинг для заказчика — точное текстовое совпадение (максимальное пересечение текста запроса с опубликованными объявлениями), без семантического поиска в MVP.

## WhatsApp Cloud API

- Входящее сообщение пользователя открывает бесплатное 24-часовое окно (Session Window).
- Вне окна общение возможно только через платные утвержденные Template Messages; сессионное сообщение вне окна не отправляется (проверка на нашей стороне, до обращения к API).
- Отправить можно только шаблон, утверждённый Meta; статусы шаблонов приходят вебхуком и видны в реестре шаблонов в админке.
- Template Message используется: при заявке заказчика поставщику с истекшим окном; при 30-дневном опросе актуальности.
- Геолокация — **только текстом** («Шымкент, центр»). Геометки WhatsApp (Attachment Location) в MVP не обрабатываются.
- Ограничения интерактивных сообщений: до 3 кнопок, до 10 элементов списка.

## Заявки и конфликты

- Техника **не блокируется** при отправке заявки: заказчик может разослать заявки нескольким поставщикам, несколько могут согласиться. Конфликт в MVP решается людьми по телефону.
- Уведомление поставщику о заявке содержит кнопки [Согласиться] / [Отказаться].

## Сценарии (конструктор)

- Сценарий — визуальная схема (граф): блоки-шаги на холсте, соединенные линиями переходов; иерархию меню администратор собирает в визуальном редакторе без разработчиков (эталон — [watbot.kz](https://watbot.kz/images/slider/1.webp)). Детали: [modules/bot-constructor.md](modules/bot-constructor.md).
- У бота несколько сценариев: главный диалог (триггер — входящее сообщение) и run-based сценарии на события — «Новая заявка», «Истекает объявление». Каждое событие создаёт отдельный изолированный запуск, не трогающий основной диалог контакта.
- Кнопки сообщений запуска несут уникальный непрозрачный токен; ответ сопоставляется только по токену (не по тексту кнопки) и попадает в тот запуск и ту опубликованную версию, которые отправили сообщение. Республикация не перекраивает уже отправленные кнопки.
- Оркестрация уведомлений — в сценариях; жёсткие ограничения Meta (24-часовое окно, только утверждённые шаблоны), статусные инварианты сущностей и окончательность решений остаются в домене и не обходятся сценарием.
- Точка входа — единственный блок «Старт»; его выход должен быть подключен. Новый диалог (первое сообщение, конец ветки или более 24 часов тишины) начинается со «Старта».
- Выходы блока: отдельный выход на каждую кнопку/элемент списка; «Продолжить» — переход по умолчанию; «Любая другая фраза» — нераспознанный ввод (если выход не подключен — бот повторяет текущий шаг); у блока «Условие» — «Да»/«Нет»; у блока «WhatsApp-сообщение» с заданным сроком ожидания — «Таймаут».
- Изменения схемы применяются только после публикации сценария; публикация блокируется при нарушении лимитов WhatsApp, неподключенных выходах кнопок/элементов и блоках, недоступных триггеру сценария. Каждая публикация фиксирует неизменяемый снимок версии.
- При публикации измененного сценария активные сессии не обрываются: пользователь завершает текущий шаг и переводится на актуальную ветку; при критическом конфликте узлов — мягкий сброс в главное меню.

## Общие ограничения MVP

- Монетизация не предусмотрена.
- Управление объявлениями (правка, архив, причины отказа) — только через Web-интерфейс по CTA URL.

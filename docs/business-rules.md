# Бизнес-правила

Сводка инвариантов проекта, извлеченная из [technical-specification.md](technical-specification.md). При конфликте приоритет у ТЗ. При изменении любого правила обновить оба файла и [changelog.md](changelog.md).

## Жизненный цикл объявления

- Статусы: **Черновик → На модерации → Опубликовано / Отклонено → В архиве**.
- Черновик формирует AI; поставщик отправляет его на модерацию или правит через Web (CTA URL).
- Через Web редактируются только черновики и отклонённые объявления; сохранение сразу отправляет на модерацию, и для отправки все бизнес-поля (тип, название, категория, описание, локация, цена) обязательны. Объявление «На модерации» и архивное — только просмотр.
- Категория объявления — значение из операторского справочника категорий (CRUD в админке), а не свободный текст. AI и веб-форма выбирают только из справочника; категория, на которую ссылаются объявления, не удаляется (см. [modules/listings-lifecycle.md](modules/listings-lifecycle.md)).
- У категории есть тип («Техника»/«Услуга»), и тип объявления обязан совпадать с типом его категории: категория определяет тип. У используемой категории тип не меняется.
- Марка объявления — необязательное значение из операторского справочника марок (CRUD в админке); есть только у типа «Техника», при смене типа на «Услуга» сбрасывается. Марка, на которую ссылаются объявления, не удаляется.
- Название объявления — короткое имя, под которым объявление фигурирует в поисковой выдаче, уведомлениях и кабинетах. В чате его составляет AI (вопрос про название не задаётся); в веб-форме оно обязательно и правится поставщиком, в админке — оператором. У объявления без названия вместо него показывается категория; без категории каждое место сохраняет свой контекстный фолбэк («без названия», «объявление», «Объявление №N» — в зависимости от места).
- Публикация возможна только после одобрения оператором в админке.
- Об одобрении и об отклонении поставщик уведомляется в WhatsApp сообщением с персональной ссылкой на объявление: в открытое 24-часовое окно — сессионным сообщением с URL-кнопкой, вне окна — утверждённым шаблоном, кнопка которого присылает ссылку в ответ на нажатие. Сбой отправки не отменяет вердикт.
- Причину отклонения поставщик видит только в Web-интерфейсе (по ссылке из уведомления), не в WhatsApp — в текст уведомления она не попадает.
- В поисковом матчинге участвуют только объявления в статусе «Опубликовано» с неистекшим `expires_at`.

## 30-дневный цикл актуальности

- `expires_at` = 30 дней с момента публикации или последнего подтверждения.
- За сутки до истечения поставщику отправляется Utility Template Message с кнопками [Да, актуально] / [Нет, в архив].
- [Да] — `expires_at` продлевается на 30 дней, статус остается «Опубликовано».
- [Нет] или отсутствие ответа в течение 24 часов — объявление автоматически уходит «В архив» и исчезает из выдачи.

## AI-ассистент

- AI включается только на шагах сценария с типом «Запрос ввода» (конструктор передает управление).
- Обязательные поля объявления: тип, категория, описание, локация, цена. При нехватке данных AI дозапрашивает их.
- Категорию AI выбирает строго из справочника категорий и никогда не придумывает: не подходит ни одна — поле остаётся пустым и задаётся уточняющий вопрос. Ветке с фиксированным типом предлагаются только категории этого типа; при автоопределении распознанная категория сама устанавливает тип объявления.
- Локация сопоставляется со справочником КАТО на нашей стороне: однозначно — привязана; одноимённые места — выбор из интерактивного списка (не расходует попытку уточнения); не найдена — уточняющий вопрос в пределах лимита.
- Марку AI выбирает строго из справочника марок и никогда не придумывает: распознаётся из слов поставщика, а не по фото или модели. Поле необязательное — уточняющий вопрос про марку не задаётся, её отсутствие не мешает подтверждению и модерации.
- Название объявления AI составляет сам из сути предложения («Аренда автокрана 25 т») — это единственное поле, которое не извлекается, а формулируется. Уточняющий вопрос про название не задаётся никогда, его отсутствие не мешает подтверждению.
- **Лимит уточнений: 2–3 попытки.** При исчерпании AI создает Черновик с пустыми полями и принудительно отправляет CTA URL для ручного заполнения.
- Сложное редактирование в чате не реализуется — только через Web по CTA URL.
- Матчинг для заказчика — гибридный: точное текстовое совпадение (пересечение слов запроса с опубликованными объявлениями) с бо́льшим весом плюс семантическая близость запроса к тексту объявления; без совпавших слов объявление попадает в выдачу только при близости выше порога (пустая выдача возможна). При недоступности AI-провайдера поиск деградирует до точного текстового совпадения.
- Перед выдачей объявлений AI-поиск собирает требования заказчика — что нужно и в каком месте — уточняющими вопросами. **Лимит уточняющих вопросов: 3** (отдельный от 3 безрезультатных попыток); явное «место не важно» принимается вместо названия места. Названное место сверяется со справочником КАТО на этом же этапе; сверка терпима к близким искажениям написания (голосовая транскрибация, опечатка). Ненайденное место не игнорируется молча — бот просит написать его точнее (обычная попытка уточнения), а если лимит исчерпан и место так и не нашлось, выдача явно помечается, что показана без учёта места. По исчерпании лимита или при недоступности AI-провайдера поиск выполняется сразу по сырому тексту сообщений.

## WhatsApp Cloud API

- Входящее сообщение пользователя открывает бесплатное 24-часовое окно (Session Window).
- Вне окна общение возможно только через платные утвержденные Template Messages; сессионное сообщение вне окна не отправляется (проверка на нашей стороне, до обращения к API).
- Отправить можно только шаблон, утверждённый Meta; статусы шаблонов приходят вебхуком и видны в реестре шаблонов в админке.
- Template Message используется: при заявке заказчика поставщику с истекшим окном; при 30-дневном опросе актуальности; при уведомлении поставщика о вердикте модерации объявления вне окна.
- Локация объявления — узел справочника КАТО (область → район → село; у крупных городов — районы города) плюс необязательное текстовое уточнение адреса. Свободный текст вместо узла недопустим; геометки WhatsApp (Attachment Location) в MVP не обрабатываются. Поиск заказчика работает по поддереву локации, при пустой выдаче бот предлагает «Искать шире» на уровень выше (см. [modules/listings-lifecycle.md](modules/listings-lifecycle.md) и [modules/ai-assistant.md](modules/ai-assistant.md)).
- Ограничения интерактивных сообщений: до 3 кнопок, до 10 элементов списка.

## Заявки и конфликты

- Техника **не блокируется** при отправке заявки: заказчик может разослать заявки нескольким поставщикам, несколько могут согласиться. Конфликт в MVP решается людьми по телефону.
- Уведомление поставщику о заявке содержит кнопки [Согласиться] / [Отказаться].

## Сценарии (конструктор)

- Сценарий — визуальная схема (граф): блоки-шаги на холсте, соединенные линиями переходов; иерархию меню администратор собирает в визуальном редакторе без разработчиков (эталон — [watbot.kz](https://watbot.kz/images/slider/1.webp)). Детали: [modules/bot-constructor.md](modules/bot-constructor.md).
- У бота несколько сценариев: главный диалог (триггер — входящее сообщение) и run-based сценарии на события — «Новая заявка», «Истекает объявление». Каждое событие создаёт отдельный изолированный запуск, не трогающий основной диалог контакта.
- Кнопки сообщений запуска несут уникальный непрозрачный токен; ответ сопоставляется только по токену (не по тексту кнопки) и попадает в тот запуск и ту опубликованную версию, которые отправили сообщение. Республикация не перекраивает уже отправленные кнопки.
- Оркестрация уведомлений — в сценариях; жёсткие ограничения Meta (24-часовое окно, только утверждённые шаблоны), статусные инварианты сущностей и окончательность решений остаются в домене и не обходятся сценарием.
- Точка входа — единственный блок «Старт»; его выход должен быть подключен. Новый диалог (первое сообщение, конец ветки или более 24 часов тишины) начинается со «Старта».
- Выходы блока: отдельный выход на каждую кнопку/элемент списка; «Продолжить» — переход по умолчанию; «Любая другая фраза» — нераспознанный ввод (если выход не подключен — бот повторяет текущий шаг); у блока «Условие» — «Да»/«Нет»; у блока «WhatsApp-сообщение» с заданным сроком ожидания — «Таймаут»; у блока «Действие» с предусловием — «Выполнено» и выход неприменимости («Заявка уже решена», «Объявление уже в архиве»; не подключён — запуск тихо завершается, ветка «успеха» не выполняется).
- Изменения схемы применяются только после публикации сценария; публикация блокируется при нарушении лимитов WhatsApp, неподключенных выходах кнопок/элементов и блоках, недоступных триггеру сценария. Каждая публикация фиксирует неизменяемый снимок версии.
- При публикации измененного сценария активные сессии не обрываются: пользователь завершает текущий шаг и переводится на актуальную ветку; при критическом конфликте узлов — мягкий сброс в главное меню.

## Общие ограничения MVP

- Монетизация не предусмотрена.
- Для поставщика управление объявлениями (правка, архив, причины отказа) — только через Web-интерфейс по CTA URL, не в чате.
- Оператор управляет объявлениями в админке напрямую: создание за поставщика (черновиком), отправка на модерацию, правка бизнес-полей, удаление по одному и массово (безвозвратно, вместе с медиа и заявками). Статусные переходы — только через действия жизненного цикла, не правкой полей (см. [modules/listings-lifecycle.md](modules/listings-lifecycle.md)).
- Оператор управляет и контактами: создание заранее (окно 24 ч у такого контакта закрыто до первого входящего), правка телефона и имени профиля, удаление по одному и массово (безвозвратно, с каскадом на объявления, заявки и историю диалога). Момент последнего входящего из админки не меняется.

# Changelog бизнес-правил

Фиксируются только изменения бизнес-правил и поведения модулей, не технические правки кода (для них есть git-история).

## 2026-07-11

- Реализован визуальный конструктор сценариев (Модуль 1): страница «Сценарий бота» в админке с холстом — блоки перетаскиваются, связи создаются кликом по кружку выхода и затем по блоку-цели или протягиванием (повторное соединение переназначает, протягивание в пустоту — разрывает), есть зум, центрирование схемы и поиск по блокам; свойства блока редактируются в боковой панели, «Старт» создается автоматически и не удаляется. Черновик не влияет на пользователей до нажатия «Опубликовать сценарий»; публикация блокируется валидацией (лимиты WhatsApp: до 3 кнопок по 20 символов, до 10 элементов списка по 24; неподключенные выходы «Старта» и вариантов; пустой текст; ссылки на несуществующие блоки), недостижимые блоки дают только предупреждение. Уточнено: публикация также сохраняет черновик; «Любая другая фраза» может оставаться неподключенной. Обновлено описание модуля [modules/bot-constructor.md](modules/bot-constructor.md).
- Запущено исполнение сценариев (рантайм Модуля 1) и конвейер сообщений WhatsApp: входящее событие асинхронно обновляет контакт (имя профиля, база 24-часового окна; повторные и опоздавшие доставки окно назад не сдвигают) и передаётся бот-движку; сообщения одного контакта обрабатываются строго по одному. Движок ведёт пользователя по опубликованной схеме: цепочки текстов по «Продолжить», остановка на интерактивных блоках, текст, совпадающий с названием кнопки/элемента, приравнен к выбору, «Любая другая фраза» / повтор шага; определено правило нового диалога (первое сообщение, конец ветки или 24 часа тишины → старт с блока «Старт») и потолок в 20 автоблоков на одно входящее. Реализована отправка сессионных сообщений через Dereu (текст, кнопки до 3, список до 10). Зафиксирована точка передачи управления AI: до реализации Модуля 2 блок «Запрос ввода (AI)» пропускается насквозь по «Продолжить». Создано описание модуля [modules/ai-assistant.md](modules/ai-assistant.md); обновлены [modules/bot-constructor.md](modules/bot-constructor.md) и [modules/whatsapp-integration.md](modules/whatsapp-integration.md).
- Реализовано доменное ядро (Модуль 4): сущности «Контакт» (телефон, имя профиля, последнее входящее — база 24-часового окна), «Объявление» (тип, категория, описание, текстовая локация, цена/тариф, медиа, `expires_at`, причина отклонения) и «Заявка» (текст запроса, выбранное объявление, статус ответа поставщика). Зафиксированы допустимые переходы статусов, включая повторную отправку на модерацию из «Отклонено»; иные переходы запрещены. В админке появились очередь модерации (одобрение публикует на 30 дней, отклонение требует обязательной причины) и просмотр контактов и заявок. Медиа объявлений (фото, аудио с транскрипцией) хранятся на локальном диске приложения (решение MVP). Создано описание модуля [modules/listings-lifecycle.md](modules/listings-lifecycle.md).

## 2026-07-09

- Интеграция WhatsApp переведена на схему Dereu Hosted Embedded Signup и переописана с нуля: номер подключает администратор со страницы «Настройки WhatsApp» (одноразовые ссылки на 10 минут, подписанный возврат, автоматический выпуск API-ключа с возможностью перевыпуска), добавлено отключение номера (Dereu хранит принятые входящие ещё 30 дней). Зафиксирован инвариант «один номер WhatsApp на проект». Ручной провижининг компании консольной командой упразднён. Создано описание модуля [modules/whatsapp-integration.md](modules/whatsapp-integration.md).

## 2026-07-08

- Конструктор сценариев зафиксирован как визуальный редактор схем: сценарий — граф из блоков и связей на холсте, а не линейный список шагов. Добавлены правила: единственный блок «Старт»; выходы блока — по одному на кнопку/элемент, «Продолжить» и «Любая другая фраза» (нераспознанный ввод; без связи — повтор текущего шага); правки применяются только после публикации сценария с валидацией лимитов WhatsApp и подключенности выходов. Создано описание модуля [modules/bot-constructor.md](modules/bot-constructor.md).

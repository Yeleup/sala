# Модуль: AI-ассистент и обработка данных

Нейросеть в роли оператора по сбору данных и поискового движка. Источник истины по бизнес-логике — [ТЗ, Модуль 2](../technical-specification.md); сводка инвариантов — [business-rules.md](../business-rules.md).

> **Статус:** сбор данных поставщика реализован; поиск для заказчика — предстоит.

## Передача управления из сценария

- AI включается только на блоках «Запрос ввода (AI)» опубликованного сценария (см. [bot-constructor.md](bot-constructor.md)): бот-движок передаёт AI ход при входе в блок и каждое следующее сообщение пользователя, пока пользователь ждёт на этом блоке.
- AI удерживает ход, пока сбор данных не завершён; завершение (в том числе по исчерпании лимита уточнений) выпускает пользователя по выходу «Продолжить» блока.
- У блока настраивается задача AI (пока одна — сбор объявления поставщика) и тип объявления: техника, услуга или «AI определит по сообщениям».

## Сбор данных поставщика

- При входе в блок бот просит рассказать о предложении: текстом, голосовым сообщением или фото.
- AI извлекает из сообщений поля объявления: тип, категорию, описание, текстовую локацию, цену/тариф. Все сообщения поставщика учитываются вместе, более поздние уточняют более ранние; AI не выдумывает значения — чего нет в сообщениях, то остаётся пустым.
- Голосовые сообщения транскрибируются, транскрипт участвует в извлечении наравне с текстом; сами аудио и фото прикрепляются к черновику объявления (см. [listings-lifecycle.md](listings-lifecycle.md)).
- Нечитаемое сообщение (например, стикер) — бот просит описать предложение словами; попытка уточнения при этом не тратится.
- Если обязательного поля (категория, описание, локация или цена) не хватает, AI задаёт один уточняющий вопрос про самое важное недостающее. **Лимит уточнений: 3 попытки.** При исчерпании черновик сохраняется с тем, что удалось собрать, поставщику уходит CTA-ссылка на веб-форму для ручного заполнения, и диалог продолжается по сценарию.
- Когда всё собрано, бот показывает сводку («Трактор, Шымкент, 10000 тг/ч. Всё верно?») с кнопками «Да, отправить» и «Исправить» (названия кнопок укладываются в лимит WhatsApp — 20 символов):
  - «Да, отправить» — объявление уходит на модерацию (см. [listings-lifecycle.md](listings-lifecycle.md));
  - «Исправить» — поставщик получает CTA-ссылку на веб-правку: сложное редактирование в чате не реализуется;
  - любой другой ответ трактуется как дополнение к объявлению — данные пересобираются, и подтверждение показывается снова.
- CTA-ссылки персональные, подписанные и действуют 7 дней; в чат они приходят интерактивным сообщением WhatsApp с URL-кнопкой (cta_url), а не текстовой ссылкой. По ссылке открывается веб-кабинет поставщика с формой редактирования (см. [whatsapp-integration.md](whatsapp-integration.md), «Веб-кабинет поставщика»).

## Поиск для заказчика (предстоит)

- Матчинг — точное текстовое совпадение: максимальное пересечение текста запроса с активными опубликованными объявлениями, без семантического поиска в MVP.
- Геолокация участвует в матчинге только как текст («Шымкент, центр»).

# Модуль: Конструктор ботов и сценариев

No-code конструктор ветвящихся диалоговых сценариев WhatsApp-бота: администратор создает и меняет логику диалогов без привлечения разработчиков. Источник истины по бизнес-логике — [ТЗ, Модуль 1](../technical-specification.md); сводка инвариантов — [business-rules.md](../business-rules.md).

## Визуальный редактор схем

Сценарий создается и редактируется как схема (граф) на холсте: блоки-шаги соединяются линиями переходов, и администратор видит всю логику диалога целиком. У бота один сценарий — главный. Визуальный эталон — конструктор [watbot.kz](https://watbot.kz/images/slider/1.webp).

Возможности холста:

- добавление блока кнопкой «+ Добавить блок», перетаскивание блоков по холсту;
- создание связи протягиванием линии от выхода одного блока ко входу другого; связь можно разорвать и переназначить;
- масштабирование (зум), центрирование схемы, поиск по блокам;
- публикация сценария отдельной кнопкой (см. «Публикация»).

## Типы блоков

| Блок | Поведение | Выходы |
| --- | --- | --- |
| Старт | Точка входа. Единственный в сценарии, создается автоматически, не удаляется. | «Продолжить» |
| Текст | Отправляет сообщение и сразу переходит дальше. | «Продолжить» |
| Интерактивное меню (кнопки) | Сообщение с кнопками, до 3 (лимит WhatsApp). | Выход на каждую кнопку + «Любая другая фраза» |
| Интерактивный список | Сообщение со списком, до 10 элементов (лимит WhatsApp). | Выход на каждый элемент + «Любая другая фраза» |
| Запрос ввода (AI) | Передает управление AI-ассистенту: сбор данных, уточняющие вопросы (см. [ai-assistant.md](ai-assistant.md)). | «Продолжить» — после завершения работы AI |

Если вариантов ответа больше трех, вместо кнопок используется интерактивный список.

## Выходы и связи

- Каждый выход блока соединяется ровно с одним блоком; на вход блока может приходить любое число связей.
- Выбор кнопки или элемента списка переводит пользователя по соответствующему выходу. Текстовый ответ, совпадающий с названием кнопки/элемента, приравнивается к его выбору.
- «Любая другая фраза» — служебный выход интерактивных блоков: сюда попадает любой ввод, не совпавший ни с одним вариантом. Если выход не подключен, бот повторяет сообщение текущего шага.
- «Продолжить» — переход по умолчанию: у текстовых блоков срабатывает сразу после отправки сообщения; у блока «Запрос ввода (AI)» — после того, как AI завершил сбор данных, в том числе по исчерпании лимита уточнений (см. [business-rules.md](../business-rules.md)).

## Публикация и мягкое обновление активных сессий

- Правки схемы не влияют на пользователей, пока сценарий не опубликован: администратор редактирует черновик схемы и применяет его кнопкой «Опубликовать сценарий».
- Публикация блокируется при ошибках валидации: превышены лимиты WhatsApp (кнопки/элементы), не подключен выход блока «Старт», у интерактивного блока есть неподключенные выходы кнопок/элементов. Блоки, недостижимые от «Старта», помечаются предупреждением, но публикацию не блокируют.
- При публикации активные сессии не обрываются жестко: пользователь завершает текущий шаг и переводится на актуальную ветку; при критическом конфликте узлов (текущий шаг удален или несовместим со схемой) — мягкий сброс в главное меню.

## Исполнение опубликованного сценария (рантайм)

- Пользователей ведет только опубликованная схема; пока сценарий не опубликован, бот не отвечает на входящие.
- **Новый диалог** начинается с блока «Старт», когда: пользователь пишет впервые; предыдущий диалог дошел до конца ветки; с последнего взаимодействия прошло более 24 часов (зеркально 24-часовому окну WhatsApp).
- Получив сообщение, бот проходит цепочку блоков подряд: текстовые блоки отправляются один за другим по выходу «Продолжить», пока диалог не остановится на блоке, ожидающем ввода (меню, список, AI), или ветка не закончится.
- На интерактивном блоке нажатие кнопки/выбор элемента ведет по соответствующему выходу; текст, совпадающий с названием варианта (без учета регистра и краевых пробелов), приравнивается к выбору; прочий ввод уходит по «Любой другой фразе», а если она не подключена — бот повторяет сообщение текущего шага.
- Сообщения одного пользователя обрабатываются строго по одному, чтобы параллельные ответы не путали позицию в диалоге.
- Защита от зацикливания: за одно входящее сообщение бот проходит не более 20 блоков подряд; при превышении диалог завершается (следующее сообщение начнет новый).
- Блок «Запрос ввода (AI)» передает ход AI-ассистенту и удерживает пользователя на себе, пока AI не завершит сбор данных (см. [ai-assistant.md](ai-assistant.md)); до реализации модуля AI блок временно пропускается насквозь по «Продолжить».

# Модуль: Сущности, поля и статусы (жизненный цикл объявления)

Доменное ядро сервиса. Взаимодействие строится вокруг сущностей **«Объявление»** (от поставщика) и **«Заявка»** (от заказчика); обе привязаны к **«Контакту»** — пользователю WhatsApp. Первоисточник: [ТЗ, Модули 4 и 6](../technical-specification.md); сводка инвариантов — в [business-rules.md](../business-rules.md).

## Контакт

Пользователь WhatsApp, написавший боту. Один контакт может выступать и поставщиком, и заказчиком — роль определяется веткой сценария, а не сущностью.

- Идентифицируется номером телефона (уникален).
- Хранит имя профиля WhatsApp (может отсутствовать).
- Хранит момент последнего входящего сообщения — по нему определяется 24-часовое окно бесплатной сессии: окно открыто, если с последнего входящего прошло меньше 24 часов (см. [whatsapp-integration.md](whatsapp-integration.md)). Контакт, ещё не приславший ни одного сообщения, окна не имеет.

## Объявление

Предложение поставщика (техника или услуга). Черновик формирует AI из свободного текста, аудио и фото; все бизнес-поля, кроме типа, могут остаться пустыми (например, при исчерпании лимита уточнений AI — см. business-rules).

### Поля

- **Тип** — Техника или Услуга; известен всегда (задаётся веткой сценария до включения AI).
- **Категория** — вид предложения (кран, экскаватор, сварщик).
- **Описание** — текстовая суть предложения.
- **Локация** — только текст («Шымкент, центр»); геометки WhatsApp не обрабатываются.
- **Цена/Тариф** — ожидаемая стоимость свободным текстом («10000 тг/ч»).
- **Медиа** — фотографии и/или аудио с транскрипцией (см. ниже).
- **Причина отклонения** — заполняется оператором при отклонении; поставщик видит её только в Web-интерфейсе, не в WhatsApp.
- **Срок действия (`expires_at`)** — дата, до которой объявление актуально; устанавливается при публикации: 30 дней с момента одобрения или последнего подтверждения актуальности.

### Статусы и переходы

| Статус | Кто и когда переводит |
| --- | --- |
| Черновик | Начальный статус: сформирован AI, поставщик проверяет данные. |
| На модерации | Поставщик отправил на проверку — из «Черновика» или повторно из «Отклонено» после правки в Web. |
| Опубликовано | Оператор одобрил в админке; одновременно `expires_at` = +30 дней, прежняя причина отклонения очищается. |
| Отклонено | Оператор отклонил в админке; причина обязательна. |
| В архиве | Из «Опубликовано»: поставщик снял с публикации через Web либо истёк 30-дневный цикл без подтверждения. |

Иные переходы (например, публикация в обход модерации или архивирование черновика) недопустимы. Возврат из архива в MVP не предусмотрен — поставщик создаёт новое объявление.

В поисковом матчинге для заказчиков участвуют только объявления «Опубликовано» с неистекшим `expires_at`. Подтверждение актуальности ([Да, актуально] в 30-дневном опросе) продлевает `expires_at` ещё на 30 дней без смены статуса.

### Модерация в админке

- Очередь модерации — объявления в статусе «На модерации», старые сверху.
- **Одобрить**: объявление становится «Опубликовано», `expires_at` = +30 дней.
- **Отклонить**: оператор обязан указать причину; без причины отклонение невозможно.
- Оператор не редактирует содержимое объявления — правки делает поставщик через Web (CTA URL).

## Медиа

- К объявлению прикладываются фотографии и голосовые сообщения; для аудио сохраняется текстовая транскрипция (её использует AI и видит модератор).
- Файлы хранятся на локальном диске сервера приложения — решение MVP, внешнее облачное хранилище не используется.

## Заявка

Отклик заказчика на конкретное объявление. Техника не блокируется: заказчик может отправить заявки по нескольким объявлениям, конфликт согласий решается людьми по телефону.

### Поля

- **Текст запроса** — исходная формулировка заказчика («нужен кран 25 тонн»), по которой шёл матчинг.
- **Выбранное объявление** — вариант, выбранный из предложенного списка.
- **Статус ответа поставщика.**

### Статусы заявки

| Статус | Когда |
| --- | --- |
| Ожидает ответа | Начальный статус: уведомление поставщику отправлено. |
| Согласие | Поставщик нажал [Согласиться]. |
| Отказ | Поставщик нажал [Отказаться]. |

Решение из статусов «Согласие»/«Отказ» не меняется — повторное взаимодействие идёт новой заявкой.

## Просмотр в админке

Оператор видит списки контактов и заявок (только просмотр) и очередь модерации объявлений с карточкой объявления, включая медиа и транскрипции аудио.

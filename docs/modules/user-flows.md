# Модуль: Пользовательские сценарии

Сквозные пути поставщика и заказчика в WhatsApp-боте. Первоисточник — [ТЗ, Модуль 5](../technical-specification.md); поведение отдельных механизмов описано в профильных модулях: [bot-constructor.md](bot-constructor.md), [ai-assistant.md](ai-assistant.md), [whatsapp-integration.md](whatsapp-integration.md), [listings-lifecycle.md](listings-lifecycle.md).

## Типовой сценарий бота

Схема собирается в визуальном конструкторе; типовые варианты устанавливаются и публикуются командой `bot:install-default-scenario` (уже опубликованный сценарий перезаписывается только с `--force`, каждый оценивается отдельно). Команда ставит три сценария: главный диалог и два сценария уведомлений — «Новая заявка», «Продление объявления» (см. [bot-constructor.md](bot-constructor.md), [listings-lifecycle.md](listings-lifecycle.md)).

Структура типового главного сценария:

- **Старт** → приветствие → главное меню с тремя кнопками: **[Я поставщик]**, **[Я заказчик]**, **[Мои объявления]**. Нераспознанный ответ повторяет меню.
- **[Я поставщик]** → выбор «Что вы предлагаете?»: **[Техника]** / **[Услуга]** → AI-блок сбора объявления соответствующего типа → завершающее сообщение.
- **[Я заказчик]** → AI-блок поиска → завершающее сообщение.
- **[Мои объявления]** → блок «Мои объявления (CTA)»: боту приходит интерактивное сообщение с URL-кнопкой «Открыть кабинет» и персональной подписанной ссылкой в веб-кабинет; ветка завершается.

## Поставщик: размещение объявления

1. Пользователь пишет боту (любое первое сообщение начинает диалог со «Старта»).
2. Выбирает [Я поставщик], затем [Техника] или [Услуга] — тип объявления фиксируется веткой.
3. Рассказывает о предложении текстом, голосом и/или фото; AI собирает поля (категория, описание, локация, цена), при нехватке — до 3 уточняющих вопросов, дальше — CTA-ссылка на веб-форму (см. [ai-assistant.md](ai-assistant.md)).
4. Подтверждает сводку кнопкой «Да, отправить» — объявление уходит на модерацию; «Исправить» — CTA-ссылка на веб-правку.
5. Оператор одобряет (публикация на 30 дней) или отклоняет с причиной; причина видна только в веб-кабинете.
6. Через 30 дней приходит опрос актуальности [Да, актуально]/[Нет, в архив] (см. [listings-lifecycle.md](listings-lifecycle.md)).

## Поставщик: управление объявлениями

1. [Мои объявления] в главном меню — бот присылает персональную ссылку в веб-кабинет.
2. В кабинете: список объявлений со статусами, причины отклонения, правка черновиков и отклонённых (сохранение отправляет на модерацию), снятие опубликованного с публикации (см. [whatsapp-integration.md](whatsapp-integration.md), «Веб-кабинет поставщика»).

## Заказчик: поиск и заявка

1. Выбирает [Я заказчик]; бот просит описать, что нужно и в каком городе/районе.
2. Пишет запрос текстом («нужен кран 25 тонн, Шымкент»).
3. AI матчит запрос по опубликованным объявлениям и присылает интерактивный список до 10 вариантов; пусто — просьба переформулировать (до 3 попыток).
4. Выбор варианта создаёт заявку и уведомляет поставщика кнопками [Согласиться]/[Отказаться] (сессионно или шаблоном — по 24-часовому окну).
5. Ответ поставщика возвращается заказчику: согласие — с телефоном поставщика для связи, отказ — с предложением подобрать другие варианты. Техника не блокируется: заявки можно отправить нескольким поставщикам, конфликт согласий решается по телефону.

# Модуль: Интеграция WhatsApp (через Dereu)

WhatsApp-канал проекта работает через Dereu — официального Meta Tech Provider: Dereu хранит WABA клиента, принимает вебхуки Meta и форвардит их проекту; проект не обращается к Meta Graph API напрямую. Ограничения самого канала WhatsApp (24-часовое окно, шаблоны, CTA-переходы) описаны в [ТЗ, Модуль 3](../technical-specification.md); сводка — в [business-rules.md](../business-rules.md).

**Инвариант:** у проекта ровно один номер WhatsApp. Одна инсталляция = одна компания в Dereu = один номер.

## Подключение номера

Номер подключает администратор на странице «Настройки WhatsApp» админ-панели (`/admin/whatsapp`) без участия разработчиков и оператора Dereu.

Флоу подключения (Hosted Embedded Signup):

1. Администратор нажимает «Подключить WhatsApp» и попадает на страницу Dereu, где проходит Embedded Signup от Meta (вход в Meta Business, выбор/создание номера).
2. По завершении Dereu возвращает администратора на страницу настроек; номер привязывается автоматически.
3. API-ключ компании (нужен для отправки сообщений) выпускается сразу после подключения. Если выпуск не удался, номер остаётся подключённым, а на странице появляется предупреждение и кнопка «Перевыпустить API-ключ».

Правила и граничные случаи:

- Ссылка запуска подключения и ссылка возврата одноразовые и живут 10 минут; повторное использование или устаревание — подключение нужно запустить заново (данные не теряются).
- Ссылка возврата принимается только с корректной криптографической подписью Dereu; при ошибке подписи ничего не сохраняется.
- Повторное прохождение подключения (в т.ч. после отключения) обновляет существующую привязку, а не создаёт вторую.
- Перевыпуск API-ключа немедленно гасит предыдущий ключ.
- Для подключения нужен доступ к Meta Business и номер, не занятый другим приложением WhatsApp; если на номере стоит 2FA-PIN в Meta Business Manager, его нужно снять заранее.

Статусы привязки: **не подключён** (начальное состояние) → **подключён** → **отключён** (можно подключить заново).

Страница настроек также показывает значения для оператора Dereu: точный URL вебхука проекта и origin для списка разрешённых адресов возврата. Интеграция требует заполненных переменных окружения `DEREU_PLATFORM_KEY`, `DEREU_WEBHOOK_SECRET`, `DEREU_EXTERNAL_ID`, `DEREU_CONNECT_SECRET`, `DEREU_CONNECT_PREFIX` (значения выдаёт оператор Dereu вместе с platform key); пока они не заполнены, страница показывает предупреждение и не даёт запустить подключение.

## Приём входящих сообщений

- Dereu форвардит проекту события WhatsApp (входящие сообщения и статусы доставки: отправлено/доставлено/прочитано/ошибка) на единый URL вебхука; URL и secret задаются один раз при выпуске platform key и должны совпадать с проектным точно, включая префикс `/api`.
- Каждое событие проверяется по криптографической подписи до какой-либо обработки; события с неверной подписью отклоняются и не сохраняются.
- Дедупликация повторных доставок: входящие сообщения — по идентификатору сообщения WhatsApp (wamid), статусные события — по идентификатору события. Повтор не создаёт дубликата.
- Неизвестные типы сообщений не отбрасываются — событие сохраняется целиком для последующей обработки.
- Приёмник отвечает быстро (2xx); события сохраняются и обрабатываются асинхронно. Обработка диалогов — зона бот-движка (см. [bot-constructor.md](bot-constructor.md)).
- Входящее сообщение обновляет контакт: телефон, имя профиля (если пришло) и время последнего входящего — базу 24-часового окна. Повторные доставки и события, пришедшие не по порядку, не сдвигают окно назад. Отображаемое имя, заданное вручную, входящие не трогают (см. [listings-lifecycle.md](listings-lifecycle.md), «Контакт»).
- Сообщения одного контакта обрабатываются строго последовательно; события чужой компании (не совпал идентификатор компании Dereu) помечаются обработанными и пропускаются.
- Статусные события (отправлено/доставлено/прочитано/ошибка) пока только сохраняются; их обработка появится с трекингом исходящих.

## Отправка сообщений

- Исходящие уходят через Dereu от имени подключённого номера с API-ключом компании; пока номер не подключён или ключ не выпущен, отправка невозможна.
- Поддерживаются сессионные сообщения: текст, интерактивные кнопки (до 3 — лимит WhatsApp), интерактивный список (до 10 элементов). Бесплатная доставка — только в открытое 24-часовое окно контакта; попытка отправить сессионное сообщение вне окна пресекается локально, до обращения к Dereu.
- Вне окна доставляются только платные шаблонные сообщения (Template Messages) — исключительно утверждённые Meta шаблоны из реестра (см. ниже); переменные `{{n}}` подставляются при отправке.
- Для проактивных уведомлений (заявка заказчика, 30-дневный опрос актуальности) канал выбирается автоматически: закрыто окно — утверждённый шаблон, открыто — тот же текст шаблона бесплатным сессионным сообщением (единый источник формулировки — тело шаблона; для сессионной доставки утверждение Meta не требуется). Уведомления оркеструются сценариями конструктора — блок «WhatsApp-сообщение» задаёт канал (адаптивно / только сессионное / только шаблон), шаблон и сопоставление его переменных `{{n}}` с данными заявки или объявления (см. [bot-constructor.md](bot-constructor.md)).
- Отправка асинхронная: Dereu принимает сообщение в очередь, фактический статус доставки приходит позже вебхуком.

## Шаблоны сообщений (Template Messages)

Шаблоны живут в Meta; проект ведёт их локальный реестр на странице «Шаблоны WhatsApp» в админке.

- **Создание**: оператор задаёт имя (строчные латинские буквы, цифры, подчёркивания), язык, категорию (утилитарный/маркетинг/аутентификация), текст с переменными `{{n}}` и, при необходимости, кнопки быстрого ответа (до 3, до 25 символов). Для каждой переменной обязателен пример значения — без примеров Meta отклоняет шаблон сразу. Шаблон регистрируется в Meta через Dereu и появляется в реестре со статусом «На модерации Meta».
- **Библиотека шаблонов**: встроенный каталог стандартных шаблонов проекта — «30-дневный опрос актуальности» (кнопки [Да, актуально]/[Нет, в архив]) и «Новая заявка по объявлению» (кнопки [Согласиться]/[Отказаться]). Оператор отмечает нужные — они регистрируются в Meta без ручного ввода текстов; уже добавленные повторно не предлагаются. Отказ Meta по одному шаблону не блокирует добавление остальных.
- **Модерация**: вердикт Meta (утверждён/отклонён с причиной) приходит асинхронно вебхуком и обновляет реестр автоматически. Повторное утверждение очищает сохранённую причину отклонения.
- **Синхронизация**: кнопка «Синхронизировать с Meta» подтягивает шаблоны, созданные напрямую в Meta Business Manager, и их актуальные статусы; локальные записи шаблонов, удалённых на стороне Meta, убираются из реестра.
- **Удаление**: удаляет шаблон и в Meta, и из реестра; отправлять его становится невозможно.
- Отправить можно только шаблон в статусе «Утверждён»; редактирование шаблона не предусмотрено — Meta требует создать новый.
- События чужой компании Dereu игнорируются, повторные доставки одного события не применяются дважды.

## Веб-кабинет поставщика (управление через Web)

Сложные действия с объявлениями выводятся из чата в мобильный веб-кабинет. Попасть в него можно только по персональной подписанной ссылке (CTA URL), которую бот отправляет URL-кнопкой WhatsApp; ссылки действуют 7 дней, ссылка с неверной подписью или просроченная не открывается. Логина и пароля нет — доступ определяется самой ссылкой.

Страницы и действия:

- **«Мои объявления»** — список объявлений контакта: категория (с маркой, если указана), тип, описание, локация и цена, статус; у отклонённых — причина отклонения (только здесь, не в WhatsApp), у опубликованных — дата окончания публикации.
- **Имя поставщика** — на странице «Мои объявления» поставщик задаёт своё отображаемое имя. Имя хранится на контакте, поэтому сразу меняется во всех его объявлениях и в сообщениях бота с переменной «Контакт: имя»; пустое поле возвращает имя из профиля WhatsApp. Входящие сообщения WhatsApp заданное имя не перезаписывают (см. [listings-lifecycle.md](listings-lifecycle.md), «Контакт»).
- **Редактирование** — доступно черновикам и отклонённым объявлениям. Форма правит тип, категорию, марку, описание, локацию и цену/тариф; для отправки на проверку все эти поля, кроме марки, обязательны — веб-форма и есть место, где поставщик дозаполняет то, что AI не собрал в чате. Категория выбирается выпадающим списком из операторского справочника, сгруппированным по типу (см. [listings-lifecycle.md](listings-lifecycle.md), «Справочник категорий») — произвольный текст и категория, не совпадающая по типу с объявлением, не принимаются. Марка (только у типа «Техника», необязательна) выбирается выпадающим списком из справочника марок; при пустом справочнике поле не показывается, а при смене типа на «Услуга» марка сбрасывается (см. [listings-lifecycle.md](listings-lifecycle.md), «Справочник марок»). Локация выбирается автокомплитом по справочнику КАТО (подсказки с цепочкой «место, район, область» появляются по мере ввода); значение вне справочника не принимается, а для деталей внутри населённого пункта есть отдельное необязательное поле «уточнение адреса». Здесь же управляются фотографии: можно догрузить новые (JPG, PNG или WebP, до 5 МБ каждое) и отметить лишние к удалению — файл удаляется безвозвратно; всего у объявления не более 10 фотографий (см. [listings-lifecycle.md](listings-lifecycle.md), «Медиа»). Голосовые сообщения и их транскрипции через веб не редактируются. Сохранение сразу отправляет объявление на модерацию (для отклонённого — повторно).
- **Снятие с публикации** — опубликованное объявление переводится в архив кнопкой «Снять с публикации» (см. [listings-lifecycle.md](listings-lifecycle.md)).
- Объявление «На модерации» и архивное открываются только на просмотр — включая фотографии и транскрипции голосовых.

Ссылки, выданные до появления кабинета (на просмотр черновика), продолжают открываться — теперь они ведут на полноценную форму.

## Отключение номера

- Кнопка «Отключить номер» (с подтверждением) немедленно останавливает приём и отправку сообщений через номер: компания деактивируется на стороне Dereu, привязка получает статус «Отключён», API-ключ удаляется.
- Уже принятые входящие Dereu хранит ещё 30 дней после деактивации (полная очистка не запрашивается).
- Если компания уже деактивирована или отсутствует на стороне Dereu, локальная привязка всё равно приводится к «Отключён».

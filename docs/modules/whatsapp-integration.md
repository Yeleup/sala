# Модуль: Интеграция WhatsApp (через Dereu)

WhatsApp-канал проекта работает через Dereu — официального Meta Tech Provider: Dereu хранит WABA клиента, принимает вебхуки Meta и форвардит их проекту; проект не обращается к Meta Graph API напрямую. Ограничения самого канала WhatsApp (24-часовое окно, шаблоны, CTA-переходы) описаны в [ТЗ, Модуль 3](../technical-specification.md); сводка — в [business-rules.md](../business-rules.md).

**Инвариант:** у проекта ровно один номер WhatsApp. Одна инсталляция = одна компания в Dereu = один номер.

## Подключение номера

Номер подключает администратор на странице «Настройки WhatsApp» админ-панели (`/admin/whatsapp`) без участия разработчиков и оператора Dereu.

Флоу подключения (Hosted Embedded Signup):

1. Администратор нажимает «Подключить WhatsApp» и попадает на страницу Dereu, где проходит Embedded Signup от Meta (вход в Meta Business, выбор/создание номера).
2. По завершении Dereu возвращает администратора на страницу настроек; номер привязывается автоматически.
3. API-ключ компании (нужен для отправки сообщений) выпускается сразу после подключения. Если выпуск не удался, номер остаётся подключённым, а на странице появляется предупреждение и кнопка «Перевыпустить API-ключ».

Правила и граничные случаи:

- Ссылка запуска подключения и ссылка возврата одноразовые и живут 10 минут; повторное использование или устаревание — подключение нужно запустить заново (данные не теряются).
- Ссылка возврата принимается только с корректной криптографической подписью Dereu; при ошибке подписи ничего не сохраняется.
- Повторное прохождение подключения (в т.ч. после отключения) обновляет существующую привязку, а не создаёт вторую.
- Перевыпуск API-ключа немедленно гасит предыдущий ключ.
- Для подключения нужен доступ к Meta Business и номер, не занятый другим приложением WhatsApp; если на номере стоит 2FA-PIN в Meta Business Manager, его нужно снять заранее.

Статусы привязки: **не подключён** (начальное состояние) → **подключён** → **отключён** (можно подключить заново).

Страница настроек также показывает значения для оператора Dereu: точный URL вебхука проекта и origin для списка разрешённых адресов возврата. Интеграция требует заполненных переменных окружения `DEREU_PLATFORM_KEY`, `DEREU_WEBHOOK_SECRET`, `DEREU_EXTERNAL_ID`, `DEREU_CONNECT_SECRET`, `DEREU_CONNECT_PREFIX` (значения выдаёт оператор Dereu вместе с platform key); пока они не заполнены, страница показывает предупреждение и не даёт запустить подключение.

## Приём входящих сообщений

- Dereu форвардит проекту события WhatsApp (входящие сообщения и статусы доставки: отправлено/доставлено/прочитано/ошибка) на единый URL вебхука; URL и secret задаются один раз при выпуске platform key и должны совпадать с проектным точно, включая префикс `/api`.
- Каждое событие проверяется по криптографической подписи до какой-либо обработки; события с неверной подписью отклоняются и не сохраняются.
- Дедупликация повторных доставок: входящие сообщения — по идентификатору сообщения WhatsApp (wamid), статусные события — по идентификатору события. Повтор не создаёт дубликата.
- Неизвестные типы сообщений не отбрасываются — событие сохраняется целиком для последующей обработки.
- Приёмник отвечает быстро (2xx); события сохраняются и обрабатываются асинхронно. Обработка диалогов — зона бот-движка (см. [bot-constructor.md](bot-constructor.md)).

## Отключение номера

- Кнопка «Отключить номер» (с подтверждением) немедленно останавливает приём и отправку сообщений через номер: компания деактивируется на стороне Dereu, привязка получает статус «Отключён», API-ключ удаляется.
- Уже принятые входящие Dereu хранит ещё 30 дней после деактивации (полная очистка не запрашивается).
- Если компания уже деактивирована или отсутствует на стороне Dereu, локальная привязка всё равно приводится к «Отключён».
